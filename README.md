# WDSigEx

Parse Windows Defender extracted signature databases, enumerate threat records, export threat chunks, collect statistics, and decode selected signature types for reverse engineering and analysis.

+ This program is distributed as-is, without any warranty;
+ No official support, if you like this tool, feel free to contribute;
+ Not everything in this project is confirmed accurate. Treat data without any additional documentation as best-guess.

## Features
* Parse extracted Windows Defender signature databases and enumerate threat records;
* Resolve threat names from embedded data, optionally backed by an external catalog CSV created by MpEnum;
* Does heuristics analysis over extracted data;
* Export matched threat chunks into individual binary files;
* Filter output by threat name substring;
* Filter output by signature type name or raw signature type value;
* Generate per-threat and global signature statistics as CSV;
* Generate HTML charts for top signature distributions;
* Decode supported signature types into structured JSON/text output.

## Supported analysis capabilities
Current decode support includes structured or heuristic handling for selected signature types, including work on:

* `SIGNATURE_TYPE_AAGGREGATOR`
* `SIGNATURE_TYPE_AAGGREGATOREX`
* `SIGNATURE_TYPE_BM_INFO`
* `SIGNATURE_TYPE_FILENAME`
* `SIGNATURE_TYPE_FILEPATH`
* `SIGNATURE_TYPE_FOLDERNAME`
* `SIGNATURE_TYPE_ASEP_FILEPATH`
* `SIGNATURE_TYPE_ASEP_FOLDERNAME`
* `SIGNATURE_TYPE_REGKEY`
* `SIGNATURE_TYPE_FRIENDLYFILE_SHA256`
* `SIGNATURE_TYPE_PUA_APPMAP`
* `SIGNATURE_TYPE_DEFAULTS`
* `SIGNATURE_TYPE_STATIC`
* `SIGNATURE_TYPE_SIGTREE`
* `SIGNATURE_TYPE_SIGTREE_EXT`
* `SIGNATURE_TYPE_SIGTREE_BM`
* `SIGNATURE_TYPE_VERSIONCHECK`
* `SIGNATURE_TYPE_LUASTANDALONE`

Other signature types are preserved, enumerated and included in statistics even when no dedicated decoder exists yet.

## Usage
```text
wdsigex <FilePath> <OutputDirectory> [--catalog <CatalogCsv>] [--threat <substring>] [--sig-type <type>]
        [--quiet] [--no-export] [--stats-only] [--html] [--decode] [--decode-json]
        [--decode-only] [--decode-dir <Directory>] [--extract-lua]
```

* `FilePath` - path to an extracted Defender database or compatible input file;
* `OutputDirectory` - directory where reports, exports and generated outputs will be written;
* `--catalog <CatalogCsv>` - optional external threat catalog CSV used to resolve threat names (generated by MpEnum);
* `--threat <substring>` - filter threats by name substring, case-insensitive;
* `--sig-type <type>` - filter threats by contained signature type;
  * accepts values like `SIGNATURE_TYPE_SIGTREE_BM`
  * accepts shorthand like `SIGTREE_BM`
  * accepts raw byte values like `0xB3` or `179`
* `--quiet` - reduce console output;
* `--no-export` - do not export matched threat chunks as `.bin` files;
* `--stats-only` - generate statistics only, implies `--no-export`;
* `--html` - generate HTML charts for signature statistics;
* `--decode` - run signature decoding phase;
* `--decode-json` - run decode phase and emit JSON-formatted decode output;
* `--decode-only` - run decode phase without exporting raw threat chunks, implies `--decode` and `--no-export`;
* `--decode-dir <Directory>` - write decode outputs into the specified directory, implies `--decode`;
* `--extract-lua` - when decoding, export embedded Lua chunks from `SIGNATURE_TYPE_LUASTANDALONE` records.

## Examples
```text
wdsigex mpavbase.extracted out
wdsigex mpavbase.extracted out --catalog defender.csv
wdsigex mpavbase.extracted out --threat Sirefef
wdsigex mpavbase.extracted out --sig-type SIGNATURE_TYPE_SIGTREE_BM
wdsigex mpavbase.extracted out --sig-type 0xB3
wdsigex mpavbase.extracted out --decode
wdsigex mpavbase.extracted out --decode --decode-json
wdsigex mpavbase.extracted out --decode --extract-lua
wdsigex mpavbase.extracted out --decode-only --decode-json
wdsigex mpavbase.extracted out --quiet --stats-only
wdsigex mpavbase.extracted out --html
```

## Output
Depending on options, the tool can generate:

* `output.txt` - listing of parsed threats;
* `missing.txt` - catalog entries not found in parsed data;
* `ThreatsStats.csv` - per-threat signature type counts;
* `ThreatsGlobalStats.csv` - global signature type counts;
* `Top30GlobalStatsChart.html` - top global signature counts chart;
* `ThreatGroupStatsCharts.html` - grouped signature charts by threat family prefix;
* exported threat chunk files as individual `.bin` blobs;
* decode outputs in text and/or JSON form;
* extracted Lua chunks under a dedicated output subdirectory when `--extract-lua` is enabled.

## Notes
+ This tool operates on extracted / unpacked Defender database material, not on original packed VDM containers;
+ Threat names are resolved from embedded file records first, then optionally enriched from external catalog data;
+ When both `--threat` and `--sig-type` are used, both filters must match for a threat to be included;
+ Signature type filtering is useful for reducing output volume and speeding up research workflows;
+ `SIGNATURE_TYPE_LUASTANDALONE` records may contain embedded compiled Lua chunks; these can be exported for separate inspection;
+ Many signature type decoders are heuristic by design and intended for reverse engineering support, not authoritative engine reproduction.

## Typical workflow
A practical research workflow looks like:

1. Parse the extracted database;
2. Filter by threat name or signature type;
3. Export only matched threat chunks;
4. Run decode on the reduced set;
5. Inspect CSV/HTML statistics for clustering;
6. Iterate on parser support for interesting signature families.

Examples:

```text
wdsigex mpasbase.extracted out --sig-type SIGTREE_BM
wdsigex mpasbase.extracted out --threat Alureon --decode --decode-json
wdsigex mpasbase.extracted out --sig-type LUASTANDALONE --decode --extract-lua
```

  + WDExtract -> ```mpavbase.extracted / mpasbase.extracted```
  + WSigEx -> signature analysis and per threat chunk dump
+ Can be built with modern .NET SDK / Visual Studio environments.
